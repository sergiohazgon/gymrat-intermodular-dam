# Planificación por sprints - GymRat

## Sprint 1 (20-22 abril)
Objetivo: definir el proyecto y diseñar la base de datos.

Tareas:
- Crear repositorio GitHub
- Definir idea del proyecto
- Redactar README inicial
- Crear estructura de carpetas
- Crear backlog de tareas
- Planificar sprints
- Analizar necesidades de la app
- Identificar entidades principales
- Diseñar base de datos inicial

## Sprint 2 (23-27 abril)
Objetivo: implementar la base funcional de la aplicación.

Tareas:
- Crear scripts SQL
- Insertar datos de ejemplo
- Implementar clases modelo
- Crear menú principal
- Configurar JDBC
- Consultar ejercicios
- Buscar ejercicios por grupo muscular
- Crear y consultar rutinas

## Sprint 3 (28 abril - 1 mayo)
Objetivo: completar integración y documentación técnica.

Tareas:
- Diseñar XML
- Diseñar XSD
- Validar XML
- Mejorar estructura del código
- Redactar documentación técnica
- Crear manual de uso
- Añadir capturas
- Redactar informe de sistemas

## Sprint 4 (2-3 mayo)
Objetivo: revisión final y entrega.

Tareas:
- Revisar el proyecto completo
- Mejorar README
- Ordenar carpetas
- Añadir capturas
- Comprobar scripts y funcionamiento
- Preparar entrega final
